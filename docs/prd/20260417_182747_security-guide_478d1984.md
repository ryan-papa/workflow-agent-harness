# PRD: SECURITY 가이드 + rp-init sops 자동화

**ID:** 20260417_182747_security-guide_478d1984
**날짜:** 2026-04-17
**상태:** Draft

## 문제·맥락

| 항목 | 내용 |
|------|------|
| 배경 | 신규 프로젝트 초기화 시 시크릿(.env) 주입·암호화 절차가 문서화되지 않아 매번 직전 프로젝트를 참조하거나 재구성 필요 |
| 원인 | sops + age 기반 시크릿 운영 규칙이 문서화되지 않음. `rp-init` 스킬에 sops 단계가 빠짐 |
| 왜 지금 | naver-cafe-monitor에서 sops 도입 완료. 다음 프로젝트 전에 표준화 안 하면 또 애드혹 세팅 |

## 타깃 사용자와 사용 맥락

| 항목 | 내용 |
|------|------|
| 대상 사용자 | Claude 에이전트 (`rp-init` 스킬 실행 중) |
| 사용 맥락 | 새 레포 생성 직후, 시크릿 세팅 단계에서 가이드 문서를 Read → 지시대로 파일 생성·암호화 수행 |
| 핵심 동기 | 결정 트리 없이 "다음 명령 한 줄"이 바로 나오게. 에이전트가 해석 여지 없이 실행 가능 |

## 핵심 시나리오

| # | 장면 | 사용자 행동 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 새 프로젝트 초기화 | `/rp-init` 실행 | 에이전트가 security-guide 참조 → `.sops.yaml` + `.env.example` 생성 |
| 2 | 시크릿 값 채우기 | 사용자가 `.env` 파일에 값 입력 | 에이전트가 `sops -e .env > .env.enc` 실행 후 `.env` gitignore |
| 3 | 신규 기기 추가 | 사용자가 새 기기 공개키 전달 | 에이전트가 `.sops.yaml` 업데이트 + 기존 `.env.enc` 재암호화 |

**실패 모드:**

| 코드경로 | 시나리오 | 감지 방법 | UX 대응 |
|----------|----------|----------|---------|
| sops 암호화 | age 키 없음 | `sops -e` 에러 메시지 | "age-keygen -o ~/.config/sops/age/keys.txt 먼저 실행" 안내 |
| sops 암호화 | `.sops.yaml` 규칙 미매칭 | `no matching creation rules found` | 규칙 path_regex 예시 출력 |
| sops 복호화 | 공개키 미등록 기기 | `Failed to get the data key` | 기기 공개키를 관리자에게 제출 → `.sops.yaml` 업데이트 절차 안내 |
| .env 커밋 | 평문 누출 | `.gitignore`에 `.env` 없음 | `rp-init`이 `.gitignore` 자동 추가 |
| 키 파일 위치 | macOS에서 sops가 키 못 찾음 | 복호화 실패 | `~/Library/Application Support/sops/age/`에도 복사 지시 |

## 대안 탐색 (A/B/C/D)

| 대안 | 요약 | 장점 | 단점 | 공수 | 판단 근거 유형 |

| D (현상 유지) | 매번 수동 세팅 | 0공수 | 반복 작업, 휴먼에러 | 0h | — |

**선택 근거:** 주 독자가 에이전트이므로 "결정 여지"를 최소화해야 자동화가 안정적. 템플릿이 단일 진실 원천(SSOT) 역할.
