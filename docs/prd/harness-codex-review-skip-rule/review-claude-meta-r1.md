## 개선 제안
- L17 스킵 트리거에 **구체 매칭 패턴 표**를 추가: `/codex:review --wait` stderr 또는 exit code 메시지에서 매칭할 키워드/정규식 리스트(예: `rate.?limit`, `token.*(limit|exceed|exhaust)`, `context.length`, `not (yet )?supported`, `quota`). 패턴 출처는 `codex` CLI 실제 출력 1건 이상 인용/링크.
- 스킵 판정 로직을 "AND 조건"으로 정의: (a) exit code ≠ 0 (또는 0이라도 stderr에 명시 패턴) AND (b) 패턴 리스트 1개 이상 매칭. 매칭 0건 → 비-스킵=중단.
- hang 처리: `--wait`의 자체 타임아웃에 의존하는지, 별도 wall-clock 타임아웃을 두는지 명시. 타임아웃 발생 시 "비-스킵=중단" 분기를 결정 사항 표에 추가.
- 다국어/버전 변경 대응: 패턴 미스매치가 반복될 때의 회수 절차(예: 회고에서 패턴 갱신 제안) 1줄 추가.
- 대안 탐색 섹션 신설: (1) 1회 재시도 후 스킵, (2) 사용자 수동 승인 후 스킵, (3) 항상 차단 유지(현행) 3안 비교 후 채택안의 근거 명시.
- 검증 항목에 **동작 시뮬레이션** 추가: 가짜 stderr(`rate limit exceeded`)로 더미 실행 → 스킵 헤더 자동 생성 여부 + 매칭되지 않는 stderr(`network unreachable`) → 워크플로우 중단되는지 양방향 검증.
- L19 증거 파일 형식에 **타임스탬프**(UTC ISO 8601) 항목 추가 — 감사·회고 추적 강화.
- L21 회고 단순 언급을 "스킵 누적 N건 이상 시 패턴 재검토 제안" 임계 조건과 연결.

> 참고: 사용자가 Q3(재시도 없음)·Q4(회고 단순 언급만)를 명시 결정했으므로, R2 보강 시 결정 자체는 유지하되 **결정 근거(대안 비교 표)**와 **운영 단서(매칭 패턴·hang 타임아웃·시뮬레이션 검증)**를 명시화한다. 회고 누적 임계는 사용자 결정과 충돌하므로 채택하지 않는다.
