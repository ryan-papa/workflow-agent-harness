# Harness: 디자인 원칙

> UI가 있는 프로젝트(웹/앱)에 적용. 토스 디자인 철학 기반.

## 1. Simplicity

- 한 화면 한 목적. 배우지 않아도 직관적으로 이해 가능
- 기능 추가는 최후의 선택(Minimum Features)
- 정책·조건이 복잡하면 UI로 극복 불가 → 정책 자체 단순화

## 2. 컬러 시스템

**시맨틱 토큰 구조:**
```
Primitive (원시값) → Semantic (의미 부여) → Component (적용)
```

**필수 시맨틱 컬러:**

| 용도 | 역할 |
|------|------|
| Primary | 주요 액션, CTA 버튼 |
| Success | 성공 피드백 |
| Error | 에러, 위험 |
| Warning | 경고, 주의 |
| Disabled | 비활성 상태 |
| Background | 배경 |
| Text / Text-light | 본문 / 보조 텍스트 |

- CSS 변수로 관리(다크모드 대응 가능)
- 프로젝트 특성에 맞는 팔레트 선택(아이용=밝고 컬러풀, 비즈니스=깔끔 미니멀)

## 3. UX 라이팅

| 규칙 | 내용 |
|------|------|
| 문체 | 해요체 통일 |
| 문장 | 능동형, 긍정형 우선 |
| 길이 | 핵심 메시지 한 줄 |
| 단어 | 모든 사용자가 이해하는 단어 (전문 용어 지양) |
| 톤 | 제안형 (강요/공포감 금지) |
| 버튼 | 다음 화면에서 일어날 일을 예측 가능하게 |

**콘텐츠 유형별 텍스트 정책:**

| 유형 | 마크다운 | 예시 |
|------|:-------:|------|
| UI 텍스트 (버튼, 라벨, 메시지) | ❌ 금지 | 순수 텍스트, HTML 태그로 강조 |
| 콘텐츠 페이지 (MDX, 블로그) | ✅ 사용 | `**굵게**`, `코드` 등 마크다운 렌더링 |
| 카카오톡 등 외부 메시지 | ❌ 금지 | 순수 텍스트, 이모지로 구분 |

- UI 텍스트 강조는 `<strong>`·`<em>` 또는 CSS로 처리
- 마크다운 미렌더링 환경에서 `**`·`##` 원문 노출 주의

**에러 메시지 원칙 (Navigating Error):**
- 무슨 일이 일어났는지
- 왜 일어났는지
- 어떻게 해결할 수 있는지
- 사용자 탓이 아닌 시스템 관점으로

## 4. 인터랙션

| 항목 | 원칙 |
|------|------|
| 전환 | 매끈하고 자연스러운 애니메이션 |
| 터치 | 시각적 피드백 (색상 변화, 스케일) |
| 로딩 | 사용자를 기다리게 하지 않음 (No More Loading) |
| 피드백 | 성공/실패 즉시 시각적으로 전달 |

## 5. 접근성

- 터치 타겟 최소 48px
- 텍스트 대비비 충분히 확보
- 디지털 비숙련 사용자도 사용 가능
- 질문형 버튼명 지양(혼란 유발)

## 6. 컴포넌트

- 재사용 가능한 컴포넌트 단위 설계
- 일관된 스타일(같은 버튼은 어디서든 같은 모양)
- 점층적 구조: 타입 → 내부 요소 → 상세 스펙

## 7. 웹 개발 체크리스트

**캐싱·배포:**

| 항목 | 내용 |
|------|------|
| 정적 자산 버저닝 | CSS/JS 파일에 해시 또는 버전 파라미터 (`style.css?v=abc123`) |
| 캐시 무효화 | 파일 변경 시 빌드 도구의 content hash 활용 |
| CDN 캐시 | Cache-Control 헤더 적절히 설정 |

**성능:**

| 항목 | 내용 |
|------|------|
| 이미지 최적화 | WebP/AVIF, lazy loading, width/height 명시 |
| 폰트 로딩 | `font-display: swap`, 서브셋, preload |
| 번들 크기 | 불필요한 의존성 제거, 트리 셰이킹 |
| 초기 로딩 | Critical CSS 인라인, JS defer/async |

**SEO·메타:**

| 항목 | 내용 |
|------|------|
| meta 태그 | title, description, og:image 페이지별 설정 |
| 시맨틱 HTML | header/nav/main/article/footer 적절히 사용 |
| sitemap | 정적 사이트는 sitemap.xml 생성 |

**보안:**

| 항목 | 내용 |
|------|------|
| 보안 헤더 | X-Content-Type-Options, X-Frame-Options |
| 외부 스크립트 | CDN 리소스에 integrity 속성 |
| HTTPS | 배포 시 HTTPS 필수 |
