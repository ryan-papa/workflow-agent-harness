# rp-qa

프로젝트 유형에 따라 QA 또는 콘텐츠 검수를 수행한다.

## 트리거

- 전체 태스크 개발 완료 후 자동
- `/rp-qa` 명령

## 유형 판별

| 유형 | 판별 기준 | QA 방식 |
|------|----------|---------|
| 코드 프로젝트 | src/ 또는 실행 코드 존재 | 기능 QA + E2E + 디자인 QA |
| 콘텐츠 프로젝트 | MDX/MD 콘텐츠 중심 | 빌드 + 렌더링 + 내용 품질 검수 |

## 코드 프로젝트 QA

### 사전 검증
- [ ] 전체 태스크가 통합 브랜치에 머지 확인
- [ ] 전체 테스트 통과 확인

### 기능 QA
- PRD F-XX 기준 시나리오 테스트
- 웹: Playwright E2E (모든 페이지 접근, 네비게이션 링크 검증)

### 디자인 QA (UI 프로젝트)
타이포그래피 / 컴포넌트 렌더링 / 여백·정렬 / 색상·대비 / 반응형 / 인터랙션 / 접근성
- 각 10점 만점, 평균 >= 8.0 + 각 항목 >= 7 → 통과
- **UI 변경 시 브라우저 QA 필수** — dev 서버 기동 후 실제 브라우저에서 확인
- 서버 미실행 환경인 경우 `⚠️ 서버 미실행으로 브라우저 QA 생략` 명시 기재
- → 상세: [`../harness-design.md`](../harness-design.md)

## 콘텐츠 프로젝트 검수

| 단계 | 내용 | 기준 |
|------|------|------|
| 빌드 검증 | `npm run build` 정상 완료 | 에러 0건 |
| 브라우저 렌더링 | dev 서버에서 변경 페이지 실제 확인 | 레이아웃 깨짐 없음 |
| 맞춤법 | 오탈자, 띄어쓰기 검수 | 발견 시 자동 수정 |
| 일관성 | 문체, 이모지 사용, 제목 형식 통일 | 기존 글과 동일 패턴 |
| 용어 통일 | 동일 개념에 다른 용어 사용 여부 | 용어사전 기준 일치 |

## 실패 시

Dev Agent 재투입 → 수정 → QA 재실행 (최대 3회, 실패 시 사용자 보고)

## ▶ 자동 전환

통과 즉시 `✓ [8] QA 통과` 출력 후 **`/rp-code-review` 자동 진입**.

→ QA 상세: [`../harness-qa.md`](../harness-qa.md)
