# rp-workflow

전체 하네스 워크플로우 오케스트레이터.

## 트리거

- 사용자가 새 기능/프로젝트를 요청할 때
- `/rp-workflow` 명령

## 역할

각 단계의 스킬을 순서대로 호출하고, 단계 간 전환 조건을 관리한다.

## 플로우

| 순서 | 스킬 | 자동 전환 조건 | 중단 조건 |
|:----:|------|--------------|----------|
| 1 | `/rp-init` | 초기화 완료 | — |
| 2 | `/rp-specify` | 질문 완료 | — |
| 3 | `/rp-prd` | 문서 작성 완료 | — |
| 4 | `/rp-plan-review` | 점수 통과 | 3회 실패 → 사용자 보고 |
| 5 | `/rp-eng-review` | 점수 통과 | 3회 실패 → 사용자 보고 |
| 6 | `/rp-task` | 분해 완료 | — |
| 7 | `/rp-dev` | 전체 태스크 완료 | 빌드/테스트 실패 |
| 8 | `/rp-qa` | QA 통과 | 3회 실패 → 사용자 보고 |
| 9 | `/rp-code-review` | 평균 >= 8.0 | 3회 실패 |
| 10 | 산출물 보고 | **사용자 승인** | 사용자 거부 |
| 11 | `/rp-ship` | 배포 완료 | CI 실패 |
| 12 | `/rp-retro` | 회고 완료 | — |

## 상태 메시지

- 진입: `▶ [N] 단계명...`
- 완료: `✓ [N] 단계명 완료`

## 규칙

- QA([8]), 코드리뷰([9]) **생략 불가**
- 산출물 보고([10]) 없이 배포([11]) 진행 **금지**
- 회고([12]) **생략 불가** — 배포 완료 후 반드시 수행
- 각 스킬의 상세 규칙은 해당 스킬 파일 참조

→ 워크플로우 상세: [`../harness-workflow.md`](../harness-workflow.md)
