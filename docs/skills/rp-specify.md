# rp-specify

도메인 → 기술 구체화. 질문을 통해 요구사항을 명확히 한다.

## 트리거

- 프로젝트 초기화 완료 후
- `/rp-specify` 명령

## 절차

1. 사용자 요청을 분석해 도메인 영역 파악
2. CLAUDE.md 질문 형식에 따라 1개씩 질문 진행
   - 질문 수: 최소 2개, 최대 15개
   - 각 질문마다 객관식 4개 + 추천 점수 + 주관식
3. 기술 스택, 아키텍처, 제약사항 확인
4. 재귀적 구체화: 새 기능 발견 시 동일 절차 반복
5. 필요 시 리서치 수행 → `docs/research/` 저장

## 질문 형식

CLAUDE.md의 Interaction 규칙을 따름:
- 1개씩 대화형 진행
- 객관식 4개 + 추천 점수(10점 만점) + ★ 추천 표시
- 주관식 항목 별도 제공

## 완료 조건

- 모든 기능이 구체화될 때까지 질문 반복
- 기술 스택·아키텍처·제약사항 확정
- 리서치 필요 시 결과 저장 완료

## ▶ 자동 전환

완료 즉시 `✓ [2] 구체화 완료` 출력 후 **`/rp-prd` 자동 진입**.

→ PRD 작성: [`rp-prd.md`](rp-prd.md)
