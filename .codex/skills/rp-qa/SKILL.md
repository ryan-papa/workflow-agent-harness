# rp-qa

프로젝트 유형에 따라 QA 또는 콘텐츠 검수 수행.

## 트리거

- 전체 태스크 개발 완료 후 자동
- `/rp-qa` 명령

## 유형 판별

| 유형 | 판별 기준 | QA 방식 |
|------|----------|---------|
| 코드 프로젝트 | src/ 또는 실행 코드 존재 | 기능 QA + E2E + 디자인 QA |
| 콘텐츠 프로젝트 | MDX/MD 콘텐츠 중심 | 빌드 + 렌더링 + 내용 품질 검수 |

## 코드 프로젝트 QA

### 사전 검증
- [ ] 전체 태스크가 통합 브랜치에 머지 확인
- [ ] 전체 테스트 통과 확인

### 기능 QA
- PRD F-XX 기준 시나리오 테스트
- 웹: **Playwright E2E 필수** (모든 페이지 접근, 네비게이션 링크 검증, 인증/가드 플로우)
- E2E 테스트가 없는 UI 태스크는 완료로 간주하지 않음

### 디자인 QA (UI 프로젝트)
타이포그래피 / 컴포넌트 렌더링 / 여백·정렬 / 색상·대비 / 반응형 / 인터랙션 / 접근성
- 각 10점 만점, 평균 >= 8.0 + 각 항목 >= 7 → 통과
- **UI 변경 시 브라우저 QA 필수** — dev 서버 기동 후 실제 브라우저 확인
- **접근성 자동 검사 필수**: `axe-core` / `@axe-core/playwright` 로 WCAG 위반 0건 확인
- 서버 미실행 환경은 `⚠️ 서버 미실행으로 브라우저 QA 생략` 명시 기재 (다음 배포 직전 반드시 재수행)
- → 상세: [`../harness-design.md`](../harness-design.md)

## AI 기능 QA (LLM·에이전트 포함 시 필수)

PRD `AI 기능 검증` 섹션이 있으면 다음을 수행:

| 항목 | 합격선 |
|------|--------|
| 정답 샘플셋 평가 | 정확도 ≥ 목표치 |
| AI 채점 기준 적용 | 평균 ≥ 8.0, 각 항목 ≥ 7 |
| 프롬프트 주입 방어 | 방어 성공 100% |
| 경계·악성 입력 테스트 | 크래시·누수 0건 |
| 재현성 | 동일 입력 3회 채점 편차 ≤ 1점 |
| 비용·응답시간 | PRD 비기능 목표 이내 |

프롬프트 변경 시 정답 샘플셋 전체 재평가 필수.
→ 상세: [`../harness-qa.md`](../harness-qa.md)

## 콘텐츠 프로젝트 검수

| 단계 | 내용 | 기준 |
|------|------|------|
| 빌드 검증 | `npm run build` 정상 완료 | 에러 0건 |
| 브라우저 렌더링 | dev 서버에서 변경 페이지 실제 확인 | 레이아웃 깨짐 없음 |
| 맞춤법 | 오탈자, 띄어쓰기 검수 | 발견 시 자동 수정 |
| 일관성 | 문체, 이모지 사용, 제목 형식 통일 | 기존 글과 동일 패턴 |
| 용어 통일 | 동일 개념에 다른 용어 사용 여부 | 용어사전 기준 일치 |
